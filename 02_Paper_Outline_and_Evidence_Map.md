## 详细大纲

### 1. 引言（~1,200 字）

**1.1 研究背景**
- 工业4.0与智能仓储发展趋势，AGV集群在电商、制造等领域的广泛应用
- 异构AGV（不同载重、速度、能耗、功能）协同调度的现实需求与挑战
- 传统运筹学方法（精确算法、元启发式）的局限：求解慢、泛化差、难以应对动态变化

**1.2 研究动机**
- AI方法（DRL、NCO）在VRP问题上取得突破性进展 [1][15][16]
- 然而现有NCO模型对训练分布高度敏感：约束紧度变化导致性能急剧退化 [8]
- 仓储场景天然存在多分布特性：不同仓库布局、任务密度、AGV配比构成不同的分布
- 现有工作缺乏面向仓储AGV场景的跨分布泛化调度研究

**1.3 核心贡献**
1. 提出**MEHA（Multi-Expert Heterogeneous Attention）**框架——首个面向仓储异构AGV集群的多专家泛化集中式调度方法
2. 设计约束紧度感知门控机制（TGate），实现跨场景分布的自适应专家路由
3. 设计异构类型注意力机制（HeteAttn），区分同类型和跨类型AGV的协调关系
4. 在标准网格仓储仿真场景中验证跨规模、跨分布泛化能力的显著提升

### 3. 问题建模（~1,200 字）

**3.1 场景描述**
- Kiva类网格仓库：离散网格地图，包含货架区、通道、充电站、拣选站
- 异构AGV集合：不同类型对应不同属性（最大载重、移动速度、电量容量、能耗率）
- 任务集合：取货/补货/移库等任务，每个任务有起点、终点、需求量

**3.2 MDP形式化定义**

- **状态空间 S**：全局网格占用矩阵 + 每台AGV的位置、电量、载重状态 + 所有任务的状态（待分配/执行中/已完成）
- **动作空间 A**：每步为每台AGV选择下一个目标节点（任务点或空闲点），集中式输出联合动作
- **状态转移 P**：确定性的网格移动 + 任务完成/电量消耗更新
- **奖励函数 R**：
  - 负的总行驶距离（主目标）
  - 负的任务完成时间（makespan）
  - 冲突惩罚：AGV间距离 < 安全距离时施加惩罚
  - 电量不足惩罚
- **约束**：
  - 每台AGV容量约束
  - 每台AGV电量约束
  - 网格通道方向约束
  - 碰撞避免约束

| 3.2 MDP | 状态/动作/奖励 | S, A, P, R |
| 3.2 约束 | 容量/电量/碰撞 | C₁-C₄ |

### 4. 方法（~2,800 字）

**4.1 总体框架概述**
- MEHA框架的整体架构图
- 输入→编码器→多专家门控→解码器→输出的数据流
- 与标准AM/POMO架构的差异对比

**4.2 异构感知编码器（Heterogeneous-Aware Encoder）**

*4.2.1 节点嵌入层*
- 任务节点嵌入：位置坐标 + 需求量 + 任务类型
- AGV节点嵌入：位置坐标 + AGV类型one-hot + 当前载重 + 当前电量
- 类型嵌入（Type Embedding）：可学习的AGV类型向量

*4.2.2 异构注意力机制（HeteAttn）*
- 标准多头注意力的扩展
- 将注意力头分为两组：
  - **同类型注意力头**：只计算同类型AGV/任务节点间的attention
  - **跨类型注意力头**：计算所有类型间的attention
- 通过注意力掩码实现分组

*4.2.3 约束紧度编码器（Tightness Encoder）*
- 输入场景全局特征：AGV数量、任务密度、平均容量利用率、平均电量约束紧度
- 通过小型MLP编码为紧度嵌入向量 `z_tight`
- 该向量作为后续门控模块的条件信号

**4.3 多专家门控模块（Multi-Expert Gating）**

*4.3.1 专家网络设计*
- K 个并行的Expert子网络（结构相同的轻量前馈网络）
- 每个Expert在不同分布范围的数据上预训练：
  - Expert₁：低任务密度场景
  - Expert₂：高任务密度场景
  - Expert₃：高AGV异构度场景
  - Expert₄：紧容量约束场景
  - ...

*4.3.2 门控网络（TGate）*
- 输入：约束紧度嵌入 `z_tight`
- 结构：2层MLP + Softmax
- 输出：K维权重向量 `w = [w₁, w₂, ..., w_K]`
- 专家输出加权融合：`h_fused = Σ w_k · Expertₖ(h_enc)`

*4.3.3 训练方案*
- 阶段1：各Expert在对应分布上独立预训练
- 阶段2：联合训练门控网络 + 冻结/微调各Expert
- 多分布采样策略：每个训练batch从不同分布中均匀采样

**4.4 集中式解码器（Centralized Decoder）**

*4.4.1 Context Embedding*
- 全局上下文：栅格图全局嵌入 + 待完成任务池嵌入
- 车辆上下文：每台AGV的当前状态嵌入
- 两者拼接形成解码器的query向量

*4.4.2 动作选择*
- Cross-Attention：query与所有候选节点嵌入计算注意力分数
- 输出所有AGV对候选节点的联合选择概率
- 推理时：贪婪解码或束搜索（Beam Search）

*4.4.3 可行解保证*
- 容量掩码：过滤超出当前AGV容量的任务节点
- 电量掩码：过滤超出续航范围的任务节点
- 冲突掩码：避免多AGV同时选择同一节点

| 子节 | 技术组件 | 创新点 |
|------|----------|--------|
| 4.2.2 | HeteAttn | 同类型/跨类型注意力分组 |
| 4.2.3 | Tightness Encoder | 紧度编码作为条件信号 |
| 4.3.1 | Multi-Expert | 分布特化专家网络 |
| 4.3.2 | TGate | 约束紧度感知动态路由 |
| 4.4.3 | 可行解掩码 | 仓储约束的集成 |

### 5. 实验（~2,500 字）

**5.1 实验设置**

*5.1.1 仿真环境*
- 网格仓库尺寸：10×10～30×30
- AGV类型：3种（小型/中型/大型），参数见附表
- 任务生成：随机生成取货/补货任务，任务密度可调节

*5.1.2 数据分布设置*
- 训练分布 D_train：中等任务密度、均衡AGV配比、标准仓库布局
- 测试分布：
  - D_test1：同分布（与训练一致）
  - D_test2：高任务密度
  - D_test3：低任务密度
  - D_test4：极端AGV配比（全部大型AGV）
  - D_test5：不同仓库尺寸（15×15, 25×25）
  - D_test6：不同AGV数量（10/20/30台）

*5.1.3 基准方法*
- **传统方法**：LKH3（VRP标准求解器上界）、OR-Tools Routing Solver
- **DRL方法**：AM (Kool et al.)、POMO、DACT [5]
- **消融变体**：
  - MEHA w/o Expert（无多专家，仅HeteAttn）
  - MEHA w/o HeteAttn（无异构注意力）
  - MEHA w/o TGate（固定权重门控）

*5.1.4 评估指标*
- 总行驶距离（TD）
- 任务完成时间（Makespan）
- 最优间隙（Gap to LKH3）
- 跨分布退化率（Degradation Rate）：`(Gap_test - Gap_train) / Gap_train`

**5.2 主要结果**

*5.2.1 同分布性能*
- 与AM、POMO、LKH3的对比表格
- 预期：在训练分布上与POMO持平或略优

*5.2.2 跨分布泛化性能*
- 六大测试分布的退化率对比表格
- 预期：MEHA在各分布上的退化率显著低于AM/POMO
- 重点展示极端分布（D_test4, D_test5）上的优势

*5.2.3 跨规模泛化*
- 不同仓库尺寸和AGV数量下的性能曲线
- 预期：MEHA在规模变化时性能下降更平缓

**5.3 消融实验**

*5.3.1 各模块贡献分析*
- 逐一移除Expert、HeteAttn、TGate的性能变化
- 分析各模块对不同分布的作用

*5.3.2 专家数量影响*
- K = 2, 4, 6, 8 时的性能对比
- 门控权重的可视化：展示不同分布下各专家的激活模式

*5.3.3 推理效率*
- 各方法的推理时间对比
- MEHA vs 传统方法的求解时间差异

| 实验 | 核心问题 | 预期结论 |
|------|----------|----------|
| 5.2.1 同分布 | 基础性能是否可比 | 持平或略优 |
| 5.2.2 跨分布 | 泛化能力是否提升 | 退化率显著降低 |
| 5.2.3 跨规模 | 规模变化适应性 | 性能下降更平缓 |
| 5.3.1 消融 | 各模块贡献 | 多专家 > 门控 > HeteAttn |
| 5.3.2 K值 | 最优专家数 | K=4～6 |

### 6. 讨论与展望（~800 字）

**6.1 结果讨论**
- 多专家机制的泛化原理：分布特化 + 动态路由避免负迁移
- 异构注意力的作用：同类型协调优于无差别注意力
- 方法的适用边界：专家数量与训练成本的权衡

**6.2 局限性**
- 仿真环境与真实仓库的差距（动态障碍、通信延迟等未建模）
- 专家数量的选择依赖经验，缺乏自适应机制
- 训练成本较高（多阶段 + 多专家）

**6.3 未来工作**
- 引入在线持续学习，实现新分布的增量适应
- 扩展至分布式MARL框架，提升大规模可扩展性
- 结合模型可解释性技术（如Probing [17]），分析专家决策逻辑
- 在真实仓储平台进行部署验证

## 论据链（Claim-Evidence Chain）

```
Claim 1 (核心主张): MEHA框架通过多专家门控机制解决跨分布泛化问题
  ├── Evidence 1.1: 多专家模块使模型在6种不同分布上的退化率均低于AM/POMO (实验5.2.2)

  └── Evidence 3.1: 同分布测试中与POMO性能持平 (实验5.2.1)
```
